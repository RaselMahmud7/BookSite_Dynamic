ভূমিকা (Introduction) শূন্য থেকে Python

প্রোগ্রামিং শেখা মানে শুধু কোড লেখা নয়, প্রোগ্রামিং মানে হলো ভাবতে শেখা। একটি সমস্যাকে ছোট ছোট ধাপে ভেঙে, কম্পিউটারকে খুব পরিষ্কার করে বলা কী করতে হবে। “প্রথমে এটা করো, তারপর ওটা করো, শেষে ফলাফল দেখাও।”

কম্পিউটার আমাদের মতো বাংলা/ইংরেজি ভাষা সরাসরি বোঝে না। কম্পিউটার ভিতরে ভিতরে বোঝে শুধু ০ আর ১ (অন/অফ)। তাহলে আমরা কীভাবে কম্পিউটারকে কাজ বলবো? এজন্যই আছে প্রোগ্রামিং ভাষা যেটা মানুষ সহজে লিখতে পারে, আর কম্পিউটার সেটা বুঝে কাজ করে।

এই বইতে আমরা Python ব্যবহার করেছি কারণ Python শেখা সহজ, syntax পরিষ্কার, আর নতুনদের জন্য perfect। তুমি একদম শুরু থেকে ধাপে ধাপে শিখবে ভ্যারিয়েবল, ডেটা টাইপ, অপারেটর, কন্ডিশনাল, লিস্ট, লুপ, ফাংশন, এবং শেষ পর্যন্ত ছোট ছোট প্রজেক্ট তৈরি করতে পারবে।

শুরু করার আগে কমেন্টস কেন গুরুত্বপূর্ণ

কমেন্ট হলো এমন লেখা যেটা Python রান করে না শুধু মানুষ পড়ার জন্য থাকে। ভালো কমেন্ট কোডকে readable এবং maintainable করে। তুমি কয়েক মাস পরে নিজের কোড দেখলেও যেন বুঝতে পারো অথবা অন্য কেউ দেখলেও যেন দ্রুত বুঝতে পারে এটিই কমেন্টের কাজ।

কমেন্ট কেন দরকার
  • কোডের উদ্দেশ্য দ্রুত বোঝাতে
  • জটিল লজিক বা ফর্মুলা ব্যাখ্যা করতে
  • ভবিষ্যতে নিজে বা টিমে কাজ করা সহজ করতে
  • TODO বা NOTE লিখে রাখতে পরে কী উন্নতি করবে

১) Single line comment (#)

এক লাইনের কমেন্ট লিখতে # ব্যবহার হয়। লাইনের শুরুতে লিখতে পারো আবার একই লাইনে code এর পরেও লিখতে পারো inline comment হিসেবে।

single_comments.py
Code
# This is a single-line comment
x = 10  # This is an inline comment
print(x)
OUTPUT
10

২) Multi line comment বা Docstring

Python এ আলাদা করে block comment নেই। তবে অনেক লাইন লিখতে চাইলে """ ... """ ব্যবহার করা যায়। বিশেষ করে function বা class এর শুরুতে থাকলে এটাকে Docstring বলে কারণ এটা function কী কাজ করে সেটা ডকুমেন্ট করে।

docstring.py
Code
def add(a, b):
    """
    Adds two numbers and returns the result.

    Parameters:
        a (int/float): first number
        b (int/float): second number
    Returns:
        int/float: sum of a and b
    """
    return a + b

print(add(5, 3))
OUTPUT
8

৩) TODO এবং NOTE

কোড লিখতে লিখতে অনেক সময় মনে হবে পরে এটা ঠিক করবো বা পরে উন্নতি করবো। তখন TODO বা NOTE লিখে রাখা ভাল অভ্যাস।

todo_note.py
Code
# TODO: Add input validation later
# NOTE: This is a simple example for beginners

price = 100
tax = 0.15
total = price + (price * tax)
print("Total:", total)
OUTPUT
Total: 115.0

৪) Good vs Bad comments

  • ভালো কমেন্ট কেন লেখা হয়েছে উদ্দেশ্য বা লজিক বোঝায়
  • খারাপ কমেন্ট obvious জিনিস বলে যেমন x = 10 # x is 10
  • কমেন্ট প্রয়োজন মতো দাও কোডও যেন পরিষ্কার থাকে
  • কোড বদলালে কমেন্টও আপডেট করো outdated কমেন্ট সমস্যা করে

Common Mistakes

  • কমেন্ট দিয়ে গুরুত্বপূর্ণ code ঢেকে রাখা পরে সরিয়ে ফেলো বা ঠিক করো
  • কোড বদলানোর পর পুরনো কমেন্ট আপডেট না করা
  • অতিরিক্ত কমেন্ট লিখে কোড জটিল করে ফেলা
অনুশীলন
  1. একটা ছোট প্রোগ্রাম লেখো যেখানে ৩টা ভ্যারিয়েবল থাকবে এবং প্রতিটা অংশে দরকারি কমেন্ট দাও
  2. একটা add() ফাংশন বানিয়ে docstring লিখো parameters এবং returns সহ
  3. তোমার আগের কোনো কোড দেখে ৩টা TODO লিখে রাখো পরে কী উন্নতি করবে
